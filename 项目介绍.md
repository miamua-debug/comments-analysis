# 产品洞察分析系统 — 项目介绍

## 项目简介

产品洞察分析系统是一站式电商数据智能分析平台，聚合京东、天猫、抖音、小红书四大平台数据，通过 LLM 驱动的语义分析引擎，自动生成用户评论洞察报告、品牌产品策略报告和品类趋势分析报告。核心创新在于将多平台异构数据源统一接入、AI 隐式打标替代人工标注、端到端纯前端+本地代理的零部署架构，使品牌运营人员无需编程即可在数分钟内完成从数据采集到专业分析报告的全流程，效率提升 200 倍以上。

## 一、核心痛点

电商品牌和商家面临三个核心数据难题：

1. **评论数据碎片化**：用户评论分散在京东、小红书等多个平台，缺乏统一的采集和分析工具。手动整理耗时且遗漏关键信息，品牌无法系统性地从海量评论中提取用户画像、核心卖点、潜在痛点等结构化洞察。

2. **产品策略分析依赖人工**：品牌需手动梳理竞品店铺的 SKU 矩阵、价格带分布、型号配置差异，效率极低且难以量化。一个中型店铺 200+ SKU 的分析，传统方式需要数据分析师 3-5 个工作日。

3. **品类趋势缺乏数据支撑**：小红书等种草平台的笔记内容蕴含品类趋势信号（用户关注点、情感倾向、品牌策略），但缺乏工具化的采集和分析能力，品牌无法基于数据做内容策略决策。

## 二、关键创新点

1. **多平台数据聚合**：首创将京东（API直连）、抖音/天猫（Apify商业采集）、小红书（opencli浏览器登录态）三大电商生态的数据源统一接入同一平台，打破数据孤岛。

2. **AI驱动的隐式打标**：区别于传统规则关键词提取，系统通过 LLM 对每条评论进行 8 个维度（人群、场景、产品、质量、服务、体验、市场、情感）的语义级隐式打标，即使评论中未明确提及也能基于上下文推断，标注覆盖率远超规则方案。

3. **Markdown + GitHub 渲染的报告引擎**：AI 输出标准 Markdown 格式，前端采用 marked.js + github-markdown-css 渲染，支持可折叠章节、KPI 指标卡、情感色块、引用卡片等交互式报告体验，兼顾可读性和专业性。

4. **端到端离线架构**：纯前端 + 本地代理服务器，无需后端部署，API Key 本地存储，数据直连 AI 服务，零第三方中转。

## 三、价值贡献

**效率提升**：单店铺 200+ SKU 的产品策略分析从 3-5 天压缩至 10 分钟（SKU获取约2分钟 + AI分析约8分钟），效率提升 **200 倍以上**。

**成本降低**：基于 DeepSeek API（成本约为 GPT-4 的 1/50），单次评论洞察分析（50条评论）成本约 ¥0.02，年处理 10 万条评论的总成本控制在 ¥50 以内。

**体验改善**：品牌运营人员无需编程技能，通过可视化界面即可完成数据获取→AI分析→交互式报告查看的全流程，操作门槛降低至零。

**收入增长**：通过策略分析报告中的价格带定位、产品矩阵优化、竞品对标建议，品牌可优化 SKU 定价和产品线规划，辅助提升店铺转化率和客单价。

## 四、方案介绍

系统采用「浏览器前端 + 本地代理 + AI 云服务」三层架构：

```
┌─────────────────────────────────────────────────────┐
│              浏览器前端 (index.html)                  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐             │
│  │ 评论洞察  │ │ 产品策略  │ │ 品类趋势  │  ← 3大模块  │
│  │ ·获取评论 │ │ ·SKU获取 │ │ ·笔记获取 │             │
│  │ ·分析报告 │ │ ·策略报告 │ │ ·趋势报告 │             │
│  └──────────┘ └──────────┘ └──────────┘             │
├─────────────────────────────────────────────────────┤
│            本地代理 (serve.ps1)                       │
│  ┌──────────────────────────────────────────┐       │
│  │  /api/proxy → DeepSeek API (AI分析)       │       │
│  │  /api/fetch-reviews → JD评论 (PowerShell)  │       │
│  │  /api/fetch-store-skus → JD店铺 (PS/搜索)  │       │
│  │  /api/fetch-douyin-skus → 抖音 (Apify)     │       │
│  │  /api/fetch-tmall-skus → 天猫 (Apify)      │       │
│  │  /api/fetch-xhs-notes → 小红书 (opencli)   │       │
│  └──────────────────────────────────────────┘       │
├─────────────────────────────────────────────────────┤
│              数据源 & AI                             │
│  JD.com │ 天猫 │ 抖音 │ 小红书 │ DeepSeek API        │
└─────────────────────────────────────────────────────┘
```

**核心技术选型**：纯前端 SPA（HTML+CSS+JS），无框架依赖；marked.js + github-markdown-css 渲染报告；Chart.js 数据可视化；Lucide 图标库；SheetJS Excel 解析；本体代理基于 .NET HttpListener + PowerShell/Python 子进程实现多平台数据采集，通过 SSE 流式推送进度和结果。
